## What to build

Create a real-device mobile QA checklist and record baseline observations for Safari on iPhone and Chrome on Android. The output should make later layout and interaction changes evidence-based rather than speculative.

## Acceptance criteria

- [ ] QA checklist covers small portrait phone, tall portrait phone, and landscape phone.
- [ ] Checklist covers maze readability, D-pad reachability, page scrolling, held direction behavior, pause, restart, and desktop keyboard regression.
- [ ] Safari iPhone observations are recorded.
- [ ] Chrome Android observations are recorded.
- [ ] Findings are translated into clear follow-up decisions for sizing, landscape, held state, restart safety, and test needs.
